"Regular expression" window

  • Regular expressions are specified using the following syntax:
    L :==   σ for σ ∈ {a-z,B-Z}
          | A meaning the alphabet
          | L1L2 (concatenation)
          | L1-L2 (difference)
          | L1+L2 (union)
          | (L)^n for n in {1, ..., 9} (n-fold concatenation)
          | (L)* (Kleene Star)
    The input expression is syntax-checked on each change (green: correct, red: error).

    Examples: (ab+cd)*aA*
  • By default, the alphabet is detected automatically, taking every letter appearing in the regular expression. This can be tweaked using the following box:
    Alphabet:
    autodetect
    When a word is specified therein, the alphabet becomes the set of letters of the word (thus "aab" and "ab" result in the same alphabet). The special word "autodetect" restores the default behavior.
  • Computations are done "on the fly," meaning that all computations will be updated every time a new valid regular expression is entered. This can be deactivated with the following checkbox:
    Compute on the fly:
    Click on to force an update.
  • The "Computations" menu offers the following options:
    • Minimal automaton: compute and display the minimal automaton
    • Syntactic monoid: show the Egg Box diagram of the syntactic monoid of the language (see Pin's book, Chapter V).
    • Syntactic order: display the syntactic order of the language (see Pin's book, Section IV.5.3).
    • Computation of the Σk-pairs of syntactic monoid: compute some pairs of the monoid, relying on the tightness property. If the monoid is shown as tight, then the set of pairs is complete. Otherwise, some pairs might be missing. The pairs that make the tightness property fail at each level are shown in red. (See this article for a presentation of tightness)

"Automaton" window

  • Automata are specified using the following syntax:
    Transitions:
    state.σ -> state_1, state_2, ..., state_n
           where σ ∈ {a-z,B-Z}
           and state is any string.
    state.A -> state_1,state_2,...,state_n
           where A stands for the alphabet
    Initial states:
    state_1, ...., state_n
    Final states:
    state_1, ...., state_n
  • By default, the alphabet is detected automatically, taking every letter appearing on a transition. This can be tweaked using the following box:
    Alphabet:
    autodetect
    When a word is specified therein, the alphabet becomes the set of letters of the word (thus "aab" and "ab" result in the same alphabet). The special word "autodetect" restores the default behavior.

  • Computations are done "on the fly," meaning that all computations will be updated every time a new valid regular expression is entered. This can be deactivated with the following checkbox:
    Compute on the fly:
    Click on to force an update.
  • The "Computations" menu offers the following options:
    • Draw automaton: draw the input automaton (green state: initial, double state: final). To download the picture in SVG, right click and "Save As."
    • Minimal automaton: compute and display the minimal automaton
    • Transition monoid: show the Egg Box diagram of the monoid generated by the transitions (see Pin's book, Section IV.3). Does not require initial/final states to be specified.
    • Syntactic monoid: show the Egg Box diagram of the syntactic monoid of the language (see Pin's book, Chapter V).
    • Syntactic order: display the syntactic order of the language (see Pin's book, Section IV.5.3).
    • Computation of the Σk-pairs of transition monoid: compute some pairs of the monoid, relying on the tightness property. If the monoid is shown as tight, then the set of pairs is complete. Otherwise, some pairs might be missing. The pairs that make the tightness property fail at each level are shown in red. (See this article for a presentation of tightness)
